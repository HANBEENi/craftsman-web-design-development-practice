# body 안 주요 태그 정리

레이아웃을 나눌 때 자주 사용하는 태그들을 정리한 메모입니다.  
특히 웹디자인개발기능사 실기처럼 화면을 구역별로 빠르게 나눠야 할 때, 각 태그의 의미를 알고 있으면 구조를 더 깔끔하게 잡기 좋습니다.

## 먼저 기억할 점

- 화면을 나누는 태그는 크게 `div`와 시맨틱 태그로 생각하면 편함
- `div`는 의미 없는 일반 박스
- `header`, `main`, `section`, `footer` 같은 태그는 의미가 있는 박스
- 시험에서는 빠른 구현이 중요하므로 `div`만으로도 작업은 가능함
- 다만 구조를 더 명확하게 이해하고 정리하려면 시맨틱 태그를 같이 알아두는 것이 좋음

## div

### 무엇인가

`div`는 가장 기본적인 블록 태그입니다.  
특별한 의미는 없고, 단순히 화면을 나누거나 묶을 때 사용합니다.

### 주로 언제 쓰는가

- 레이아웃 구역을 크게 나눌 때
- 여러 요소를 하나로 묶고 싶을 때
- CSS 적용 범위를 묶고 싶을 때
- 특별한 의미가 필요 없는 박스를 만들 때

### 메모

가장 많이 쓰는 태그입니다.  
시험에서는 속도 때문에 `div` 중심으로 먼저 구조를 잡는 경우가 많습니다.

## header

### 무엇인가

페이지 또는 특정 구역의 머리 부분을 나타내는 태그입니다.

### 주로 언제 쓰는가

- 사이트 상단 영역
- 로고와 메뉴가 들어가는 부분
- 어떤 섹션의 제목 영역

### 메모

보통 웹페이지 맨 위쪽에 많이 사용합니다.  
로고, 메뉴, 사이트명 등이 들어가는 경우가 많습니다.

## nav

### 무엇인가

다른 페이지나 다른 구역으로 이동하는 메뉴 영역을 나타내는 태그입니다.

### 주로 언제 쓰는가

- 상단 메뉴
- 좌측 메뉴
- 퀵메뉴
- 페이지 이동 링크 묶음

### 메모

메뉴가 들어가는 부분이면 `nav`를 우선 떠올리면 됩니다.  
단순 장식용 링크 몇 개를 묶는 것보다, 실제 이동 목적이 분명한 메뉴일 때 더 잘 맞습니다.

## main

### 무엇인가

문서의 핵심 본문 내용을 나타내는 태그입니다.

### 주로 언제 쓰는가

- 페이지 중앙 주요 콘텐츠 영역
- 슬라이드, 공지사항, 배너, 갤러리 등이 들어가는 중심 부분

### 메모

한 페이지에서 가장 중요한 내용이 들어가는 영역입니다.  
보통 `header`와 `footer` 사이의 중심 콘텐츠를 감쌀 때 사용합니다.

## section

### 무엇인가

주제별로 내용을 구분하는 구역 태그입니다.

### 주로 언제 쓰는가

- 공지사항 영역
- 갤러리 영역
- 배너 영역
- 바로가기 영역
- 슬라이드 영역

### 메모

내용을 주제별로 나누고 싶을 때 사용합니다.  
시험 레이아웃에서는 메인 안의 각 칸을 나눌 때 자주 사용할 수 있습니다.

## article

### 무엇인가

하나의 독립된 콘텐츠 단위를 나타내는 태그입니다.

### 주로 언제 쓰는가

- 게시글
- 뉴스 글
- 공지 글 한 개
- 카드형 콘텐츠 한 개

### 메모

`section`이 큰 구역을 나누는 느낌이라면, `article`은 그 안에 들어가는 독립된 글이나 콘텐츠 하나를 뜻하는 경우가 많습니다.

예를 들어 공지사항 전체는 `section`, 공지사항 한 줄 한 줄 또는 글 하나는 `article`로 생각할 수 있습니다.

## aside

### 무엇인가

메인 내용과 직접적인 중심 내용은 아니지만, 옆에서 보조하는 내용을 나타내는 태그입니다.

### 주로 언제 쓰는가

- 사이드바
- 광고 영역
- 관련 링크 모음
- 추가 정보 박스

### 메모

예전에는 왼쪽이나 오른쪽 보조 메뉴 영역을 만들 때 많이 떠올릴 수 있습니다.  
반드시 옆에 있어야만 하는 것은 아니고, 핵심 내용의 보조 역할이면 사용할 수 있습니다.

## footer

### 무엇인가

페이지 또는 특정 구역의 맨 아래 정보를 나타내는 태그입니다.

### 주로 언제 쓰는가

- 사이트 하단 영역
- 저작권 표시
- 하단 메뉴
- 회사 정보
- SNS 링크

### 메모

웹디자인개발기능사 레이아웃에서는 거의 항상 마지막 구역에 해당합니다.  
하단 메뉴, 주소, copyright 등을 넣는 위치로 생각하면 됩니다.

## 시맨틱 태그를 쓰는 이유

- 구조 의미를 파악하기 쉬움
- 코드를 다시 볼 때 이해가 빠름
- 협업할 때 어떤 영역인지 바로 알기 쉬움
- 문서 구조가 더 명확해짐

## 시험 기준으로 생각하면

- 빠르게 구현해야 할 때는 `div`만으로도 충분히 가능함
- 다만 연습 단계에서는 `header`, `nav`, `main`, `section`, `footer`를 같이 써보는 것이 구조 이해에 도움이 됨
- 중요한 것은 태그 이름보다도, 화면을 어떤 구역으로 나눌지 먼저 판단하는 것

## 간단한 예시 구조

```html
<body>
  <header>상단 영역</header>
  <main>
    <section>이미지 슬라이드</section>
    <section>공지사항</section>
    <section>갤러리</section>
  </main>
  <footer>하단 영역</footer>
</body>
```

## 한 줄 정리

- `div`: 의미 없는 기본 박스
- `header`: 상단 영역
- `nav`: 메뉴 영역
- `main`: 핵심 본문 영역
- `section`: 주제별 구역
- `article`: 독립된 콘텐츠 하나
- `aside`: 보조 영역
- `footer`: 하단 영역
